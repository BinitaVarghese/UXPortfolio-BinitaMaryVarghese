Wordsup - Enhance your German vocabulary!!

INTRODUCTION

The project was done as part of the UX design training at Career Foundry It was my first experience with UX designing and was executed in 4 weeks.

The project included designing a low fidelity wireframe and prototype for a native app for German vocabulary learning for busy individuals. The project also included coming up with the concept, conducting competitive market research, developing a user persona and the problem statement and user testing

The project introduced me to various techniques that are used during UX designing especially the crazy 8 technique which I find is very useful for taking quick decisions while ensuring that you have looked at alternatives.

Skills used -

  • Competitive Analysis
  • User flows and Concept sketches
  • Persona hypothesis
  • Frameworks
  • Low fidelity peper wireframe
  • Usability Testing
  • Design Documentation & Presentation

Launch an interactive prototype.

PROBLEM STATEMENT

“My user, needs a way to enhance his/her German language vocabulary that fits effectively into his/her busy schedule, while balancing his/her time between various commitments.”

We will know this to be effective and successful when he/she can use basic German sentences and post positive reviews on the app.


User research was the primary key to the solution envisioned as people had experiences with similar products and were well aware of their needs and wants in the vocabulary app.

USER INTERVIEWS

Since this was a short project and intended to begin our UX journey, I did not conduct a user survey and relied heavily on the user interviews conducted.

For this project I decided that my focus group would be the expat community who have had some experience using vocab or language apps when they first moved to Germany in the age range of 30 - 50. I used social media to contact people in the vicinity and the interviews were done using video calls.


People are always willing to learn new things, if the learning can be done in short durations as a regular activity


COMPETITIVE ANALYSIS

With so many references on the pros and cons of existing apps from the user survey, I chose three to conduct my competitive analysis. The apps were analysed based on their visibility on search, onboarding ease and functional effectiveness.

I segregated my finding into a SWOT structure to better analyse my findings.


While all the apps that I had reviewed was fairly easy to navigate, they all seemed to target different types of audience


USER PERSONA

For this project I have created only 1 user persona based on my inputs from the user research. The user persona created has all the characteristics and needs gathered from the survey.

Based on the persona user stories and job stories were created to help define the tasks to achieve the solution to the problem statement.


Personalising the problem at hand helped me to empathise better with the user requirements and enables me to provide a better solution to cater to the user persona needs.


USER FLOWS

User flows were developed to showcase two functionalities of the app. – the login sequence and adding a word to the user defined list.

Creating user flows simplified the task ahead by breaking it into steps which were easier to design during the wireframing phase.


Creating user flow is very insightful and exciting for me. As the flow is sketched it helps me envision the various possibilities of user anticipation in the flow that need to be looked at and create solutions for them.


WIREFRAMING

Within the scope of the project, the next step was to create wireframes for the 2 user flows identified.

The wireframes were created using paper and I used mobile templates to be able to estimate the size of the screens. I also used the crazy 8 method to make quick decisions on the best way to approach screen designs.

A prototype was created to help users test the effectiveness of the design.


Sketching on paper can be time consuming especially for the repetitive elements, but I still like the ability to sketch design ideas quickly. Once the designs are clear on paper, I would move them to a digital platform.


USER TESTING

A test plan and a test script are valuable resources for efficient testing. My plan included 4 test participants and they were observed during the test in in-person test scenarios.

The test results were tabulated and classified as per the Jakob Nielsen error classifications. The wireframes and the prototype were updated based on the severity of the user observations.


Observations and misunderstandings were leveled out during the updating of the wireframes. An observation by a user was not accepted as it was just one view-point and not every user had the same requirement.



A FEW LESSONS FROM THIS PROJECT​​​​​​​​​​​​​​...

  • Time Management is a very important aspect to take in consideration during user interviews for survey as well as usability testing.

  • I also realised that people may not be available at the appointed time for various reasons, so I would have to account for cancellations and plan for more interviews than less. This would ensure I have enough inputs for the phase.

  • Setting expectations for usability test is also an area that I would focus on. Test participants need to be made aware of expectations before the actual test if need be, so that they focus only on test situations rather than the look during the prototype phase. A detailed test script would also have helped me during the test phase.
